# 전화번호부 과제 설계에 대해서 기술한다. 

1. 객체 설계
- 전화번호 , 이름 을 저장한다.
public class PhoneData {
  int phoneNum; 
  String name ; 
}

ArrayList<PhoneData> phone_list = new ArrayList<PhoneData>() ; 

- 출력할 때는 010-xxxx-xxxx형태로?
- 입력시에는 "-" 없이 입력받도록.
(생각해봄직한 부분이라면 입력할 때 아래로 쭉 선택가능한 창을 넣는 것은 어떨까?) 
- 010 , 02 등의 앞부분은 그렇게 입력하도록 하고 남은 부분을 적도록 하는 거지.
- 그리고 잘못된 전화번호 판단에 대해서는 조금 고민해볼 것. 


2. 기능 설계 
2-1. 전화번호부 등록
- 이름, 전화번호를 입력받고 해당객체를 생성 후 ArrayList에 넣는다.
- 이미 있는 이름이나 전화번호인지 확인하는 부분이 필요하다.
- 파일에 저장하는 것도 고려할 것. 

2-2. 전화번호부 검색/수정
- 먼저 파일로부터 저장된 내용을 읽어온다.
- 그리고 번호 또는 이름을 받아서 일치하는 것이 있는지 검색 및 수정하도록. 

2-3. 전화번호부 목록 출력 
- 파일에 한줄씩 저장하고 그것을 읽어오는 일련의 과정이 필요하다. 


3. 화면 구성
3-1. 기본 화면 구성
- 아래쪽에는 입력 라인을 넣는다. (번호 입력 감지, 잘못된 번호에 대한 처리 필요) 
- 왼쪽에는 고정 출력할 메뉴목록을.
- 오른쪽에는 선택한 메뉴에 따라서 다르게 화면 변경이 필요하다. (아마도 덮어씌우거나 하는 듯 싶다만 찾아봐야한다.) 

- 레이아웃 하나만으로는 어려울 테고 다른 레이아웃을 내부에 넣는 식으로 해야할 듯 싶다. 
- 아래쪽은 BorderLayout을 사용해서 집어넣는다고 치고. 
- 가운데부분은 절반씩 나누어서 사용해야 하니까 GridLayout을 생각해보자. 

- 레이아웃을 어떤 식으로 쓸지 아직 감이 잘 안 잡힌다. 여러번 좀 건드려보자. 

3-2. 화면 전환 
- 내가 기억하는 바가 맞다면 화면 위에 덮어씌우는 식(?)으로 처리될 것같다만,
다른 메서드를 사용할지도 모른다. 찾아보는 편이 바람직하리라. 

3-3. 스크롤 바. (전체 목록) 
- 스크롤 + 수정이 되지않는 텍스트박스 출력이 이뤄져야 한다.
